Documentation



Command arguments marked in red are required.

Command arguments marked in yellow are optional.



Moderation Commands

Name

Premium

 Usage Description
warn False /warn <member> <reason> [duration] Warn a member
warnings False /warnings [member] View warnings for a member or yourself in a paginated session
remove_warning False /remove_warning <warning_id> Remove a warning by warning ID
ban False /ban <member> <reason> [delete_days=0] [attachment] Ban a member from the server - the [delete_days] is an optional argument that deletes the user's messages within the limit of those days
kick False /kick <member> <reason> [attachment] Kick a member from the server
delete_all_warnings True /delete_all_warnings Careful! This removes ALL warnings in the database for YOUR server! This CANNOT be undone. The purpose of this command is to clear warnings to make warning commands faster for your server.
mute False /mute <member> <reason> <duration> Mute a member - this uses Discord's Timeout feature so please remember to give Server Manager Moderate Members permissions for this to work!

Utility Commands

Name

Premium

 Usage Description
poll start False /poll start <topic> [description] [duration] [ping_role] [channel] Start a poll with an optional duration to end/close - only 5 polls allowed to be running for non-premium servers. Premium have access to unlimited polls
poll end False /poll end <poll_id> End a poll, the poll ID should be at the embed footer of the poll
stats False /stats View statistics for Server Manager - shows uptime, library version, bot version, and more
help False /help [command] Shows the help command

Admin Commands

Name

Premium

 Usage Description
config custom_reasons False /config custom_reasons [hidden=False] View custom reasons for the server
config custom_reason add False /config custom_reason add <name> <value> Add a custom reason, these can be used in moderation commands. They're kind of like snippets. Premium limit: 10. Non-premium limit: 5
config custom_reason remove False /config custom_reason remove <name> Remove a custom reason by name
config log_channel False /config log_channel <#channel> Set the server's log channel for moderation cases and other logs
config log_events False /config log_events Set the logging events that go to the log channel when dispatched (use dropdown)
config admin_roles False /config admin_roles [hidden=False] View the administrator roles for the server
config admin_role add False /config admin_role add <@role> Add a role as an administrator role, users with these roles can use configuration commands and other moderator-only commands
config admin_role remove False /config admin_role remove <@role> Remove a role as an administrator role
config moderator_roles False /config moderator_roles [hidden=False] View the moderator_roles for the server
config moderator_role add False /config moderator_role add <@role> Add a role as a moderator role, users with these roles can use moderation commands and other moderator-only commands
config moderator_role remove False /config moderator_role remove <@role> Remove a role as a moderator role
