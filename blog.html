    Server Manager uses a custom time converter so that you can provide a human-readable time argument without the hassle of any extra steps. These duration arguments are used in Moderation, Polls, Giveaways, and more. Here is a quick guide to mastering those arguments.

    Keyâ€‹

    d - days

    h - hours

    m - minutes

    s - seconds

    Examplesâ€‹

    Using 'd' (days)
    /time argument: 1d
    vNziie--used /time
    Server ManagerBot11/04/2022
    The time argument you provided was 1 day!
    Using 'h' (hours)
    /time argument: 1h
    vNziie--used /time
    Server ManagerBot11/04/2022
    The time argument you provided was 1 hour!
    Using 'm' (minutes)
    /time argument: 5m
    vNziie--used /time
    Server ManagerBot11/04/2022
    The time argument you provided was 5 minutes!
    Using 's' (seconds)
    /time argument: 30s
    vNziie--used /time
    Server ManagerBot11/04/2022
    The time argument you provided was 30 seconds!

    Hope this helps you on the fly while either starting a giveaway, poll, or issuing warnings or mutes!

    Release v1.1.0

    September 6, 2022 Â· 2 min read
    Nziie
    Core Developer

    Hey everyone, here with a new version of Server Manager. This new version includes: moderation cases, bug fixes, and small changes.

    Casesâ€‹

    Cases may make a big impact on moderation in your server. With every moderation action (warn, mute, kick, ban), a case is created with a specific ID. These cases aren't removed when a member leaves and they don' expire. Therefore, you can keep track of a user's moderation history even if they left the server. You can edit a case's reason and proof in the /case <case_id> command. Yes, you heard me correctly: you can attach proof to a case. Case proof is helpful long term if you doubt that there was actually proof for the action taken against someone.

    note

    Warning cases are a separate system so if you remove a warning, the case tied to the warning won't be removed.

    Please also note that cases can only be created and viewed by moderators.

    CommandDescription
    /case <case_id>View/manage a case - you can edit a case in this command
    /createcase <member> <reason> [action=Other][proof=image url] [duration]Create a case for a user
    /cases [member]View cases for the whole server or yourself

    Updated Warn Commandâ€‹

    The warning system now uses "token like" warning IDs and now uses an embed for a command response.

    danger

    It seems like old warnings with different warning IDs don't seem to be valid warning IDs when trying to remove the warning, but they are in the database. We are currently working on a fix for this.

    info

    Custom branded bots now support cases as of 9/11/2022

    Custom Branded Commandsâ€‹

    CommandDescription

    Suggestion Edit Commandâ€‹

    The new suggestion edit command allows users to edit their own suggestion, only once.

    CommandDescription
    /suggestions edit <suggestion_id>This will bring up an interactive proccess to edit the suggestion
    info

    At the moment, you can only edit the suggestion itself and not it's other properties.

    Suggestion Delete Commandâ€‹

    The new suggestion delete command allows the suggestion author or an administrator to delete a suggestion. Thus, deleting it from the suggestion channel and the database.

    CommandDescription

    Warning
